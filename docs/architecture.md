# Architecture

QuattroAgents is a provider-neutral agent factory that generates specialized teams for a project's specific domain and tooling. The core domain model (`domain.py`) defines agents, skills, swarms, decisions, and interview sessions as plain data contracts. A repository scan (`analysis.py`) profiles the project's languages, build systems, test frameworks, CI setup, and tooling. The canonical state lives in `.agent-factory/` as human-inspectable JSON, including project profiles, decisions (with supersession chains), interview sessions, and generated manifests with override-detection records for detecting manual edits.

The adaptive interview engine (`interview/engine.py`) orchestrates knowledge-gap detection, question planning, answer processing, and conflict detection to build up a set of `Decision`s that guide agent/skill/swarm generation. The generation pipeline (`generation/agents.py`, `generation/skills.py`, `generation/swarm.py`) consults the project profile and active decisions to select and compose agents, skills, and swarm plans (using Kahn's algorithm for dependency-free parallel scheduling). These are then rendered to provider-specific formats (Claude Code `.claude/agents/*.md`, Codex `.codex/agents/*.toml`) through adapters (`adapters/claude.py`, `adapters/codex.py`) using a hash-guarded `GeneratedFileGuard` that detects manual modifications before overwriting. A validation layer (`validation.py`) ensures consistency (no duplicate IDs, all agents have completion criteria, no dangling references, valid tool availability, grammatical agent display format).

Both the CLI (`cli.py`) and MCP server (`mcp_server.py`) share the same tool dispatch table and operate on the same canonical `.agent-factory/` state.
