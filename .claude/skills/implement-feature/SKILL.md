---
name: implement-feature
trigger: Implement a new feature or scoped change.
---

## Workflow
1. Analyze requirements and acceptance criteria.
2. Write implementation code with appropriate abstractions.
3. Add unit tests covering normal and edge cases.
4. Verify integration with existing systems.

## Inputs
- Feature specification or user story
- Acceptance criteria
- Design notes or API contracts

## Outputs
- Implemented feature code
- Unit tests
- Integration test results

## Required tools
- None declared.

## Validation criteria
- All acceptance criteria met
- Unit test coverage above 80%
- Code review approval

## Usable by
implementation-agent
