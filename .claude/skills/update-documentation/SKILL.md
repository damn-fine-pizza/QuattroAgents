## Workflow
1. Identify affected documentation sections.
2. Update docs to reflect new behavior or API.
3. Verify examples are correct and runnable.
4. Check for consistency with other docs.

## Inputs
- Behavior change or new feature
- Updated API signatures or configs
- Related codebase changes

## Outputs
- Updated documentation
- Verified examples
- Consistency check summary

## Required tools
- None declared.

## Validation criteria
- Documentation matches current behavior
- Examples tested and working
- No orphaned or outdated references

## Usable by
documentation-agent
